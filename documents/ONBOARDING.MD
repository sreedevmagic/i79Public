# i79 Engage – Onboarding & Hiring Team Experience (UX Specification)

## 📌 Objective

Create a **simple, intuitive onboarding experience** where a new user can:

* Create their **first job in under 2 minutes**
* See it **live on a career page immediately**
* Add **Hiring Manager and HR Business Partner seamlessly**
* Avoid any heavy setup or configuration upfront

---

# 🎯 Experience Philosophy

### 1. Start with Value, Not Setup

Users should **experience success first**, not configuration.

### 2. Guide, Don’t Force

Let users move forward without blocking steps.

### 3. Keep It Lightweight

Every screen should feel **fast, minimal, and obvious**.

### 4. Add Complexity Later

Only introduce advanced features when users actually need them.

---

# 🧭 End-to-End User Journey

## Step 1: Welcome Experience

### What the user sees

A clean, friendly screen:

> “Let’s post your first job in under 2 minutes”

Two clear options:

* **Quick Start (Recommended)**
* Advanced Setup (secondary)

---

### User feeling

* No overwhelm
* Clear starting point
* Confidence that this will be fast

---

## Step 2: Create First Job (Core Action)

### What the user sees

A simple form with only essential fields:

* Job Title
* Location
* Work Type
* Job Description

With an option:

* ✨ “Generate with AI”

---

### 👥 Hiring Team (Integrated, Not Separate)

Right below job details:

**Hiring Manager**

* Pre-filled as: *You*

**HR Business Partner**

* “+ Add”

---

### When user clicks “+ Add”

A small, fast interaction appears:

* Enter name
* Enter email
* Select role

Click: **Add & Assign**

---

### What happens (from user perspective)

* The person is instantly added
* A small label appears: *Invited*
* No interruption
* No waiting

---

### User feeling

* “This is easy”
* “I didn’t have to leave the screen”
* “I can keep going”

---

## Step 3: Continue → System Auto Setup

### What the user sees

A short progress screen:

> “Setting things up for you…”

With quick confirmations:

* Career page created
* Branding applied
* Job ready

---

### User feeling

* System is smart and fast
* No effort required

---

### What the user sees

A simple screen:

> “Make it yours (optional)”

Options:

Buttons:

* Continue

---

* Skipping does not block progress

### User feeling

* Control without pressure
* Freedom to skip

# 🧠 Key UX Patterns

## 1. Inline Team Assignment

* Add team members while creating the job
* No separate setup screen

---

## 2. Smart Defaults

* User is automatically the Hiring Manager
* Reduces input effort

---

## 3. Invisible Invitations

* Team members are invited in the background
* No interruption

---

## 4. Progressive Disclosure

* Show features only when needed
* Avoid overwhelming the user

---

## 5. Skippable Steps

* Branding and integrations are optional
* User stays in control

---

# ⚠️ What to Avoid

* Asking users to configure settings before creating a job
* Forcing team setup before progress
* Redirecting to different pages during onboarding
* Blocking actions for missing optional steps
* Overloading the first screen with too many fields

---

# 💡 Ideal User Experience (In One Sentence)

> “I created a job, added my team, and saw it live — all in a couple of minutes.”

---

# 🚀 Final Outcome

A user should be able to:

* Start with zero setup
* Create a job instantly
* Add key team members effortlessly
* Publish and view their career page immediately

Without ever feeling:

* Stuck
* Overwhelmed
* Forced into configuration
